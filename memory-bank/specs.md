## 一、项目概述 (Project Overview)

### 1.1 产品定位
PtIDLE 是一款集成战棋玩法的挂机游戏（Idle + Tactical Chess）。核心特色为「所玩即所造」—— 玩家通过挂机系统深度定制卡牌与装备，在公平的战棋对局中比拼策略。

### 1.2 核心愿景
- **前端变现**：纯粹、绝对公平的 3v3 战棋竞技，杜绝数值碾压
- **后端驱动**：深度挂机系统，产出竞技所需的「机制配件」与「消耗品门票」
- **核心关联**：挂机与战棋通过「材料流水线」紧密咬合，形成闭环

### 1.3 目标用户
- 休闲挂机玩家（享受数值成长乐趣）
- 硬核策略玩家（追求战棋博弈深度）
- 两者兼顾

### 1.4 平台与技术
- **平台**：Web网页端（全栈）
- **商业模式**：非商业，内部体验娱乐

## 三、核心功能 (Core Features)

### 3.1 挂机系统（工坊）

#### 3.1.1 资源采集
- **采矿**：产出铁矿石、煤炭
- **伐木**：产出原木、树液
- **草药学**：产出止血草、荧光菇
- 玩家通过派遣穿戴生产装备的英雄执行采集任务

#### 3.1.2 材料加工
- **冶炼**：矿石 + 煤炭 → 金属锭
- **木工**：原木 → 木板
- **研磨**：草药 → 草药粉末

#### 3.1.3 制造合成（服务于战棋）
- **基础卡牌**：移动卡、轻击卡等基础指令卡
- **生产装备**：矿镐、伐木斧等采集加成装备（用于挂机采集，**不带入战棋**）
- **职业认证装置**：决定棋子职业和卡牌使用权限
- **符文石**：镶嵌于卡牌上，赋予/改变**机制效果**（如击退、吸血、燃烧），而非数值提升
- **战术消耗品**：回血药、烟雾弹等

### 3.2 战棋系统（战场）

#### 3.2.1 棋子与职业（绝对公平设计）

**核心原则**：棋子为白板素体，**不存在任何数值差异**

- 棋子本身没有任何先天属性
- 通过「职业认证装置」决定职业，获得：
  - 固定基础HP（同一职业所有玩家完全相同）
  - 固定移动力（同一职业所有玩家完全相同）
  - 固定能量上限（同一职业所有玩家完全相同）
  - 职业专属卡牌使用权
- **绝对公平**：同一职业的装备不存在品质上的数值差异，没有"传奇"比"普通"血多的说法

| 职业 | HP | 移动力 | 能量上限 | 定位 |
|------|-----|--------|----------|------|
| 战士 | 高 | 低 | 3 | 坦克/近战 |
| 弓手 | 中 | 中 | 3 | 远程单体 |
| 法师 | 低 | 低 | 3 | 远程AOE/组合技 |

#### 3.2.2 卡牌系统（机制驱动）

**核心原则**：符文改变**机制**而非**数值**

- 每张卡牌拥有：名称、效果、能量消耗
- 基础卡牌：玩家通过挂机制造，为"白板指令卡"
- 符文镶嵌：向卡牌嵌入符文石，赋予/改变**机制效果**
  - 例如：伤害不变但附加击退；伤害略降但获得吸血
  - 强力的机制符文通常会增加能量消耗或附带前提条件
- **不存在数值碾压**：所有伤害与血量建立在严格的个位数或十位数模型上
- 卡牌为消耗品，使用后需重新制造
- MVP仅实现基础卡牌，不含符文镶嵌

#### 3.2.2.1 机制符文示例
| 符文 | 效果 | 代价 |
|------|------|------|
| 击退符文 | 伤害不变，将敌人推入特定位置 | 能量消耗+1 |
| 吸血符文 | 伤害略降，获得续航 | 能量消耗+1 |
| 燃烧符文 | 附加持续伤害 | 需特定站位条件 |

#### 3.2.3 回合流程
1. **蛇形激活**：双方交替激活单位
2. **抽牌**：激活单位从牌库抽3张
3. **移动**：免费移动（不超过移动力）
4. **打牌**：消耗能量打出手牌
5. **回合结束**：未使用手牌入弃牌堆，可保留1张

### 3.3 资源流转（挂机 ↔ 战棋关联）

```
采集（铁矿石/原木/草药）
    ↓
加工（铁锭/木板/草药粉末）
    ↓
制造（基础卡牌/生产装备/战术消耗品）
    ↓
PVP对战（消耗卡牌）
    ↓
结算（获得体验/成就感）
```

## 四、核心组件 (Core Components)

### 4.1 数据模型

#### 玩家 (Player)
```
- id: UUID
- username: string
- password_hash: string
- created_at: timestamp
- last_login: timestamp
- resources: JSON (铁矿石、原木、草药等)
- materials: JSON (铁锭、木板等)
- cards: JSON (玩家拥有的卡牌)
- production_gear: JSON (生产装备)
- characters: JSON (棋子信息)
```

#### 棋子 (Character)
```
- id: UUID
- player_id: UUID
- name: string
- profession: enum (warrior/ranger/mage)
- health: int
- max_health: int
- movement: int
- energy: int
- max_energy: int
- deck: array (卡牌列表)
- position: {x, y}
```

#### 卡牌 (Card)
```
- id: UUID
- name: string
- type: enum (attack/defense/tactical)
- cost: int (能量消耗)
- effect: JSON (效果描述)
- profession: enum (通用/战士/弓手/法师)
```

## 八、核心关联总结

挂机系统与战棋系统的关联是本项目的核心特色：

1. **材料流转**：挂机采集 → 加工 → 制造 → 战棋消耗
2. **职业凭证**：挂机产出「职业认证装置」→ 决定棋子职业和卡牌权限
3. **卡牌依赖**：战棋中使用的所有卡牌均来自挂机系统制造
4. **循环驱动**：PVP消耗卡牌 → 驱动玩家持续挂机获取材料 → 制造更强（机制更丰富）卡牌

### 核心设计公理

- **绝对公平**：棋子无数值差异，同职业面板完全相同
- **机制驱动**：挂机追求的是**横向**扩展（不同流派组合），而非**纵向**数值碾压
- **符文目的**：改变卡牌机制（击退、吸血、燃烧），而非增加伤害数值
- **组合克制**：通过不同机制符文的组合克制特定战术，而非用数值碾压对手
